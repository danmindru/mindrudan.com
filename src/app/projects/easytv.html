Introduction

EasyTV is a small search engine that based on selected user preferences, provides the best fitting TV packages in a certain location. Together with a responsive front-end search interface, the system requires a capable back-end for package management, locations or even individual channels.

The process

The project sets off with quick prototyping of the front and back-end. For the back-end, the database architecture and class diagram are created. For the front-end, the client identity is consolidated and adapted to the application vision. Next, the project follows a falsificationist philosophy by testing each decision and iterating until it passes the tests. Prototypes are made for each iteration.

Front-end

The design of EasyTV balances two stylistic categories: usability functionalism and digital modernism. The aim was to find a middle point between an appealing and a functional interface. A mobile first development pattern was followed for concept creation.

The approach on the search options was by the last iteration changed to an interface that resembles a complex filter. Channels are selectable from a list and the search functionality is reserved as an alternative option. The channel list is dynamically changed based on the application usage (for example, channels that are on average selected the most will appear at the top of the list).
